Задача:

1) Необходима конфигурация QA для сборки тестовой версии приложения для отправки тестерам.
2) Используется Google Ads SDK для показа рекламы в приложении. В конфигурациях Debug и QA необходимо показывать баннеры с тестовым идентификатором, чтобы, например, разработчики и тестеры случайно не нажимали на рекламу (Google может принять это за накручивание кликов и забанить рекламный аккаунт). Таким образом нужно хранить различные идентфикаторы для баннера и рекламной страницы в разных конфигурациях.
3) Используется Mixpanel SDK.  В конфигурациях QA и Release используем аналитику, поэтому нужно чтобы SDK инициализировался только в этих конфигурациях. Также нужно хранить токен для этого сервиса.
4) Использует детальное логгирование только в конфигурациях Debug и QA.

Что было сделано:

1) Создана конфигурация QA и настроена новая схема QA для сборки приложения для тестеров.
2) В Build Settings созданы User-Defined настройки GOOGLE_ADS_BANNER и GOOGLE_ADS_INTERSTITIAL. В конфигурациях Debug и QA там лежат тестовые идентфикаторы, предоставляемые Google. В конфигурации Release будут лежать реальные идентификаторы для показа рекламы. Затем в Info.plist был добавлен словарь GoogleAds с данными настройками, чтобы использовать их в коде.
3-4) В Build Settings создана User-Defined настройка MIXPANEL_TOKEN. Для конфигураций QA и Release там сохранен токен данного сервиса аналитики. В Info.plist былы добавлена строка MixpanelToken, чтобы использовать ее в коде. 
Созданы Preprocessor Macros DEBUG=1, QA=1, RELEASE=1 для соотвествующих конфигураций. В AppDelegate добавлены булевы поля detailLogs (детальное логгирование) и analytics (инициализация анлитики) которые задаются в соотвествии с макросами.
